# ExoPlanetFinder

A machine learning pipeline for detecting exoplanet transit signals in TESS light curves, using the Box Least Squares (BLS) algorithm and a Random Forest / Gradient Boosting classifier trained on real confirmed planets and verified false positives from the TESS Object of Interest (TOI) catalog.

---

## How It Works

1. **Fetch** — Downloads labeled targets (confirmed planets + false positives) from the ExoFOP-TESS / NASA Exoplanet Archive TOI catalog and retrieves available TESS sectors for each TIC ID from MAST.
2. **Train** — For each target, downloads the SPOC light curve, runs BLS across up to 3 sectors, extracts features, and trains a classifier. The best model between Random Forest and Gradient Boosting is saved automatically.
3. **Test** — Given any TIC ID, downloads its light curve, runs BLS, and classifies it as a planet candidate or false positive.

## Project Structure

```

```

---

## Pipeline Script Usage

```
./run_pipeline.sh [OPTIONS]

Options:
  -m, --max-targets N     Max number of targets to fetch and train on (default: 300)
  -t, --toi-only          Skip BLS during training — train on TOI table features only (fast)
  -s, --skip-fetch        Skip fetch step if labeled_tess_dataset.csv already exists
  -T, --test-only         Only run test.py — skip fetch and train entirely
  -h, --help              Show this help message
```

### Examples

```bash
# Full pipeline, 300 targets
./run_pipeline.sh

# Quick test run — 100 targets, no BLS (finishes in seconds)
./run_pipeline.sh --max-targets 100 --toi-only

# Already have the CSV? Skip the fetch step
./run_pipeline.sh --skip-fetch

# Already have a trained model? Just run the classifier on the test target
./run_pipeline.sh --test-only

# Large run — 1000 targets with full BLS (expect 30–60 min)
./run_pipeline.sh --max-targets 1000
```

## Running Scripts Individually

```bash
# 1. Fetch dataset (500 balanced targets — 250 planets, 250 false positives)
python fetch_tess_labeled_dataset.py --max-targets 500

# 1b. Fetch without sector lookup (just labels, instant)
python fetch_tess_labeled_dataset.py --max-targets 500 --skip-sectors

# 2. Train on full BLS features
python train_classifier.py --csv labeled_tess_dataset.csv

# 2b. Train on TOI table features only (no light curve downloads)
python train_classifier.py --csv labeled_tess_dataset.csv --toi-only

# 2c. Limit training targets
python train_classifier.py --csv labeled_tess_dataset.csv --max-targets 200

# 3. Test a specific TIC target (edit TIC_ID inside test.py)
python test.py
```

## Features Used by the Classifier

| Feature | Description |
|---|---|
| `bls_period` | Mean best BLS period across sectors (days) |
| `bls_duration` | Mean best transit duration across sectors (days) |
| `bls_depth` | Mean fractional flux drop across sectors |
| `bls_snr` | Mean BLS signal-to-noise across sectors |
| `bls_period_std` | Std of period across sectors — real planets are stable |
| `depth_over_duration` | Depth ÷ duration — sharpness proxy for real transits |
| `toi_period` | Period from TOI catalog (days) |
| `toi_depth` | Depth from TOI catalog (fractional) |
| `toi_duration` | Duration from TOI catalog (days) |
| `n_sectors` | Number of available TESS sectors |

---

## Labels

| Label | Meaning | TOI Disposition |
|---|---|---|
| `1` | Confirmed / known planet | `CP`, `KP` |
| `0` | False positive | `FP` |

Planet candidates (`PC`) are excluded from training data intentionally — their ground truth is unknown.

## Output

After training, the best model is saved to `exoplanet_rf_classifier.joblib`. The script prints:

- 5-fold cross-validation F1 scores for both models
- Which model won
- Classification report (precision, recall, F1)
- Confusion matrix
- Feature importances

---

## Notes

- The dataset is **balanced** — `--max-targets N` always samples `N/2` planets and `N/2` false positives.
- BLS is run on up to **3 sectors per target** during training and averaged, giving more stable features than a single sector.
- If BLS fails for a target (no SPOC data, download error), the TOI catalog values are used as a fallback so the sample isn't wasted.
- The quality bitmask is set to `"default"` — aggressive enough to remove bad cadences, lenient enough to keep scattered light data that doesn't affect BLS.
